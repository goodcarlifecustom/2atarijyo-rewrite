# SWELL記事装飾ルール

このファイルは、SEOリライト後の記事HTMLに対して、SWELL向けの装飾を適用するためのルールです。

対象ファイルは原則として以下です。

* articles/sample-article/rewritten.html

装飾適用後も、WordPressに貼り付け可能なHTMLとして成立するようにしてください。

---

## 0. 「この記事でわかること」リストの設置

記事全体の最初のH2見出しの直前に、「この記事でわかること」リストを設置してください。

### 設置内容

以下の要素を設置してください。

* `<p>【この記事でわかること】</p>`
* 記事内のすべてのH2見出しへのアンカーリンク付きリスト
* リストは `<ul>` と `<li>` を使う

### H2へのid付与

記事内のすべてのH2見出しには、アンカーリンクに対応するid属性を付与してください。

例：

```html
<h2 id="h2-title-1">見出しタイトル</h2>
```

既存のid属性がある場合は、原則として既存idを利用してください。
ただし、重複しているidや意味が分かりにくいidがある場合は、自然で分かりやすいidに修正してください。

### 注意点

* アンカーリンクは記事内のH2見出しに正しくジャンプできるようにしてください。
* H2の文言は勝手に変更しないでください。
* 「この記事でわかること」リストは、最初のH2の直前に1回だけ設置してください。

---

## 1. H2配下のH3アンカーリスト設置

必要な場合、H2見出しの導入文直後に、そのH2配下にあるH3見出しへのアンカーリンク付きリストを設置してください。

### 対象

以下のようなケースでは、H3アンカーリストを設置してください。

* H2配下にH3が3つ以上ある
* 比較、手順、方法、注意点、スポット紹介など、読者が項目移動しやすい方がよい箇所
* H2配下の情報量が多く、一覧性を高めた方がよい箇所

### 設置位置

対象H2の直下にある導入文の直後に設置してください。

### H3へのid付与

各H3見出しには、アンカーリンクに対応するid属性を付与してください。

例：

```html
<h3 id="heading-example">見出しタイトル</h3>
```

既存のidがある場合は、原則として既存idを利用してください。

---

## 2. リストのボックス装飾

記事内の重要なリストは、SWELLのキャプションボックスで囲んでください。

ただし、すべての`<ul>`を機械的に囲むのではなく、読者にとって重要なリストを優先してください。

### 優先して装飾するリスト

* 「この記事でわかること」リスト
* H2配下のH3アンカーリスト
* 手順リスト
* 注意点リスト
* 比較ポイントのリスト
* メリット・デメリットのリスト
* アフィリエイト導線に関係する重要リスト

### 使用するHTML

リスト本体は `div.cap_box_content` の中に配置してください。

```html
<!-- wp:loos/cap-block {"className":"is-style-onborder_ttl"} -->
<div class="swell-block-capbox cap_box is-style-onborder_ttl"><div class="cap_box_ttl"><span>ここにタイトル</span></div><div class="cap_box_content"><!-- wp:list -->
<ul class="wp-block-list">
<li>リスト1</li>
<li>リスト2</li>
<li>リスト3</li>
</ul>
<!-- /wp:list --></div></div>
<!-- /wp:loos/cap-block -->
```

### ボックスタイトルの付け方

リスト内容に合わせて、自然なタイトルを付けてください。

例：

* この記事でわかること
* 〇〇のポイント
* 〇〇の注意点
* 〇〇の手順
* 〇〇の比較ポイント
* 〇〇がおすすめな人
* 〇〇で失敗しないコツ

### 注意点

* タイトルはリスト内容と一致させてください。
* すでにSWELL装飾済みのリストは、二重に装飾しないでください。
* 通常の短い箇条書きまで過剰に装飾しないでください。
* HTML構造が崩れないようにしてください。

## 3. 本文の段落調整

各H2、H3見出しに対応する本文で、1つの`<p>`タグが長すぎる場合は、意味のまとまりごとに複数の`<p>`タグへ分割してください。

### 分割の目安

以下のような区切りで分割を検討してください。

* 導入
* 詳細説明
* 具体例
* 注意点
* 結論

### 注意点

* 文章の意味や流れは変えないでください。
* 極端に短い段落を量産しないでください。
* 1段落は2〜3文程度を目安にしてください。
* マーカー適用前に段落調整を行ってください。

---

## 4. マーカーの適用

段落調整後、各H2・H3見出しに対応する本文の中から、最も重要な箇所にマーカーを適用してください。

### 適用数

原則として、1つの見出しブロックにつき1箇所までにしてください。

ただし、以下のようにポジティブな重要箇所とネガティブな重要箇所が明確に両方ある場合は、最大2箇所まで適用しても構いません。

* メリット
* 注意点
* デメリット
* 失敗回避
* 重要な判断基準

### ポジティブな内容のマーカー

メリット、良い点、おすすめ、結論、重要ポイントには以下を使用してください。

```html
<span class="swl-marker mark_yellow">強調したい文章</span>
```

### ネガティブな内容のマーカー

デメリット、注意点、課題、リスクには以下を使用してください。

```html
<mark style="background-color:rgba(0, 0, 0, 0)" class="has-inline-color has-swl-deep-01-color">強調したい文章</mark>
```

### 既存マーカーの扱い

装飾を適用する前に、既存のマーカーは一度すべて削除してください。

削除対象の例：

```html
<span class="swl-marker mark_yellow">...</span>
<mark style="background-color:rgba(0, 0, 0, 0)" class="has-inline-color has-swl-deep-01-color">...</mark>
```

既存マーカーを削除したうえで、本文の意味に合わせて改めて適切な箇所へマーカーを適用してください。

---

## 5. 装飾適用後の確認

装飾後、以下を必ず確認してください。

* H2アンカーリンクが正しく設置されている
* H2のidが重複していない
* H3アンカーリンクを設置した場合、H3のidが重複していない
* 「この記事でわかること」リストが最初のH2直前に1回だけ設置されている
* キャプションボックスのHTML構造が崩れていない
* 不要な二重装飾がない
* 段落分割で意味が変わっていない
* マーカーが過剰に使われていない
* WordPressに貼り付け可能なHTMLになっている
